# Project Jeokru 시스템 기획서

## 1. 게임 개요

Project Jeokru는 탑뷰 시점의 실시간 유닛 지휘·공장 운영 게임이다.

플레이어는 마우스로 여러 아군 유닛을 선택하고 이동, 전투, 공장 작업 명령을 내린다. 공장에서는 철과 네 종류의 약을 생산할 수 있으며, 생산한 자원으로 공장을 설치·수리하거나 아군의 전투 능력을 강화한다.

게임 내 시간은 계속 흐르며 지정된 시각마다 적 병력이 생성된다. 플레이어는 반복적으로 증원되는 적으로부터 아군과 공장을 보호해야 한다.

최종 승리 조건은 현재 존재하지 않는다. 모든 공장이 파괴될 때까지 게임이 계속되는 생존형 구조다.

## 2. 핵심 플레이 흐름

1. 아군을 채굴장이나 약 공장에 배치한다.
2. 철과 약을 생산한다.
3. 철로 공장을 설치하거나 수리한다.
4. 약으로 아군을 회복하고 강화한다.
5. 레이드에 대비해 아군을 배치한다.
6. 적을 전멸시켜 레이드에 성공한다.
7. 피해를 복구하고 다음 적 증원을 준비한다.

# 4. 시간·일차 시스템

## 4.1 시간 속성

| 속성 | 설명 |
|---|---|
| 현재 시각 | 1~60 |
| 현재 일차 | 게임이 진행된 일수 |
| 시간 증가 간격 | 현실 시간 기준 설정된 초 |
| 레이드 발생 시각 | 개발 과정에서 설정 |
| 아군 증원 시각 | 개발 과정에서 설정 |

## 4.2 시간 진행 규칙

1. 게임은 1일차부터 시작한다.
2. 현실 시간 `시간 증가 간격`마다 현재 시각이 1 증가한다.
3. 현재 시각이 60을 넘으면 1로 돌아간다.
4. 60에서 1로 돌아갈 때 일차가 1 증가한다.
5. 지정된 시각이 되면 연결된 시간 이벤트를 발생시킨다.
6. 게임 오버 및 일시정지 중에는 시간 진행을 멈춘다.

모든 시간 관련 수치의 단위는 현실 시간의 초(s)다.

## 4.3 레이드 시간 이벤트

- 지정된 레이드 시각마다 적 생성 규칙을 실행한다.
- 이미 적이 존재하더라도 기존 레이드를 종료하거나 새 레이드로 교체하지 않는다.
- 기존 레이드에 추가 지원 병력이 합류하는 형태로 처리한다.
- 레이드 성공 또는 실패 판정이 발생하기 전까지 하나의 레이드 상태가 유지된다.

## 4.4 아군 증원 이벤트

- 1일차에는 정기 아군 증원이 발생하지 않는다.
- 2일차부터 지정된 시각마다 아군 2명을 생성한다.
- 레이드 진행 및 실패 여부와 관계없이 발생한다.
- 게임 오버로 시간이 멈췄다면 발생하지 않는다.

# 5. 맵·타일 시스템

## 5.1 맵

- 맵은 정사각형 타일로 구분된 이차원 평면이다.
- 타일 한 칸의 크기는 1×1이다.
- 전체 맵 크기는 개발 과정에서 설정한다.
- 유닛은 타일 단위가 아니라 연속 좌표상에서 움직인다.
- 타일은 시설 설치, 공간 점유, 공장 작업 범위 판정에 사용한다.

## 5.2 공간 점유

- 공장과 상담실이 점유한 타일에는 유닛이 진입하거나 통과할 수 없다.
- 공장과 상담실이 점유한 타일에는 다른 시설을 설치할 수 없다.
- 유닛끼리는 같은 위치에 겹칠 수 있다.
- 유닛 간 충돌로 길을 막지 않는다.

## 5.3 경로 탐색

- 유닛은 목표 지점까지 이동 가능한 최단 경로를 사용한다.
- 공장과 상담실을 장애물로 취급해 우회한다.
- 유닛끼리는 장애물로 취급하지 않는다.
- 유닛은 목표 또는 타겟으로부터 설정된 유지 거리까지 이동한다.
- 유지 거리는 0일 수도 있지만, 근거리·원거리 전투 유닛은 별도로 설정된 0보다 큰 값을 사용한다.

## 5.4 오른쪽 생성 구역

- 맵 오른쪽 카메라 밖에 고정된 유닛 생성 구역을 둔다.
- 동적으로 생성되는 아군과 적군은 모두 이 구역에서 등장한다.
- 생성 Y좌표는 생성 가능 범위 안에서 무작위로 결정한다.
- 생성된 유닛은 생성 구역에서 맵 내부로 진입한다.

# 6. 유닛 시스템

## 6.1 유닛 속성

| 속성 | 설명 |
|---|---|
| 현재 HP | 현재 체력 |
| 최대 HP | 100 고정 |
| 이동 속도 | 초당 이동 거리 |
| 공격력 | 공격 시 사용하는 수치 |
| 방어력 | 피격 대미지 감소에 사용하는 수치 |
| 공격 쿨타임 | 공격 사이의 대기시간 |
| 공격 유형 | 근거리 또는 원거리 |
| 공격 범위 | 공격 판정을 발생시키는 범위 |
| 유지 거리 | 타겟과 유지하려는 거리 |
| 팀 | 아군 또는 적군 |
| 기본 유닛 여부 | HP 0일 때 제거 여부 |
| 현재 행동 | 대기, 이동, 작업, 전투 등 |
| 현재 타겟 | 전투·작업 대상 |
| 스트레스 | 0~100 |
| 선택 여부 | 플레이어에게 선택됐는지 |

공격력, 방어력, HP 및 대미지는 실수형으로 계산한다.

## 6.2 팀 규칙

- 아군은 플레이어가 선택하고 명령할 수 있다.
- 적군은 플레이어가 선택하거나 명령할 수 없다.
- 아군은 적군만 공격할 수 있다.
- 적군은 아군 유닛과 공장을 공격할 수 있다.
- 아군은 공장을 전투 타겟으로 설정할 수 없다.

## 6.3 기본 유닛

- 게임 시작 시 기본 아군 5명을 생성한다.
- 기본 유닛의 근거리·원거리 구성과 능력치는 개발자가 설정한다.
- 기본 유닛은 HP가 0이 되어도 게임에서 제거되지 않는다.
- HP가 0이면 활동 정지 상태가 된다.
- 활동 정지 중에는 이동, 공격, 작업을 할 수 없다.
- 적의 타겟과 공격 판정에서도 제외된다.
- 회복 약으로 HP가 0보다 높아지면 다시 활동한다.

## 6.4 비기본 아군과 적군

- 기본 유닛이 아닌 아군은 HP가 0이 되면 제거된다.
- 모든 적군도 HP가 0이 되면 제거된다.
- 제거된 유닛은 회복하거나 복귀할 수 없다.

## 6.5 행동 상태

| 상태 | 설명 |
|---|---|
| 대기 | 유효한 명령이나 타겟 없이 제자리에 있음 |
| 이동 | 지정 위치로 이동 |
| 전투 | 적을 추적하고 공격 |
| 공장 이동 | 작업 대상 공장으로 이동 |
| 공장 작업 | 공장의 생산에 참여 |
| 활동 정지 | 기본 유닛의 HP가 0인 상태 |
| 상담 중 | 스트레스 회복을 위해 게임 판정에서 제외 |
| 퇴각 | 실패한 레이드의 적이 생성 구역으로 복귀 |

자동 이동 시스템은 존재하지 않는다. 유효한 목표 위치나 타겟이 없으면 유닛은 현재 위치에서 대기한다.

# 7. 유닛 선택·명령 시스템

## 7.1 단일·다중 선택

- 선택되지 않은 아군을 좌클릭하면 선택 목록에 추가한다.
- 이미 선택된 아군을 다시 좌클릭하면 해당 유닛만 선택 해제한다.
- 여러 아군을 동시에 선택할 수 있다.
- 선택된 유닛 주변에는 선택 상태를 나타내는 색상 외곽선을 표시한다.
- 우클릭하면 모든 선택을 취소한다.
- 선택된 유닛이 없다면 유닛 명령을 내릴 수 없다.

## 7.2 범위 선택

1. 선택된 유닛이 없는 상태에서 좌클릭 드래그를 시작한다.
2. 최초 클릭 좌표와 현재 마우스 좌표를 양 끝으로 하는 직사각형을 계산한다.
3. 드래그 중 직사각형을 반투명하게 표시한다.
4. 마우스를 놓으면 직사각형 안의 아군을 선택한다.

## 7.3 입력 구분

| 현재 상태와 입력 | 결과 |
|---|---|
| 선택 없음 + 좌클릭 드래그 | 범위 선택 |
| 선택 있음 + 빈 공간 좌클릭 | 위치 이동 |
| 선택 있음 + 빈 공간 좌클릭 드래그 | 선 배치 이동 |
| 다른 아군 좌클릭 | 선택 목록에 추가 |
| 선택된 아군 재클릭 | 해당 유닛 선택 해제 |
| 우클릭 | 전체 선택 취소 |

## 7.4 위치 이동

- 선택된 모든 유닛에게 같은 목표 위치를 전달한다.
- 유닛은 목표 위치까지 최단 경로로 이동한다.
- 유닛끼리 겹치는 것을 허용한다.

## 7.5 선 배치 이동

1. 선택된 유닛이 있는 상태에서 빈 공간을 좌클릭 드래그한다.
2. 드래그 중 마우스가 지나간 경로를 반투명한 선으로 표시한다.
3. 직선과 곡선을 모두 허용한다.
4. 드래그가 끝나면 선택된 유닛을 선 전체에 균등하게 분배한다.
5. 선이 짧아도 균등 분배하며 필요하면 유닛끼리 겹치게 배치한다.

# 8. 행동·타겟 시스템

## 8.1 아군 전투

- 전투 명령을 받은 아군은 가장 가까운 적군을 타겟으로 설정한다.
- 타겟에게 공격 가능한 유지 거리까지 이동한다.
- 타겟이 제거되면 새로운 전투 명령이나 자동 전투 상태가 없는 한 대기한다.

## 8.2 적군 타겟

- 적군은 항상 전투 상태다.
- 정상적인 레이드 중에는 아군 유닛과 공장을 합쳐 가장 가까운 대상을 타겟으로 선택한다.
- 같은 거리에 여러 대상이 있다면 무작위로 하나를 선택한다.
- 현재 타겟이 제거되거나 무효 상태가 되면 새로운 가장 가까운 대상을 탐색한다.

## 8.3 피격 시 타겟 변경

- 유닛이 공격받으면 자신을 가장 최근에 공격한 적을 타겟으로 설정한다.
- 타겟 변경은 현재 진행 중인 공격 쿨타임을 초기화하거나 변경하지 않는다.
- 적이 공장을 공격하던 중 아군에게 공격받으면 해당 아군으로 타겟을 변경한다.

## 8.4 작업 중 피격

1. 작업 유닛의 작업 대상 공장과 행동을 임시 저장한다.
2. 가장 최근 공격자를 타겟으로 설정한다.
3. 해당 적과 전투한다.
4. 타겟을 쓰러뜨리면 주변에 다른 적이 있어도 기존 작업으로 즉시 복귀한다.

공장으로 복귀하기 전까지는 해당 공장의 작업자 수에 포함하지 않는다.

# 9. 전투 시스템

## 9.1 공격 쿨타임

- 공격 속도 대신 공격 쿨타임을 사용한다.
- 공격 쿨타임의 단위는 초다.
- 쿨타임이 짧을수록 더 자주 공격한다.
- 공격 직후 설정된 쿨타임을 시작한다.
- 쿨타임이 끝나야 다음 공격을 실행할 수 있다.
- 공격 쿨타임에는 개발 과정에서 설정하는 하한이 존재한다.

## 9.2 대미지 계산

```text
계산 대미지 = 공격자의 공격력 - 피격 대상의 방어력
최종 대미지 = max(0, 계산 대미지)
피격 후 HP = max(0, 기존 HP - 최종 대미지)
```

- 계산 결과가 음수라면 대미지를 0으로 처리한다.
- 하나의 공격이 여러 유닛에게 적중하면 각 대상의 방어력으로 개별 계산한다.
- 공장도 방어력을 가지며 같은 방식으로 대미지를 계산한다.
- 공장 방어력은 일반적으로 높게 설정한다.

## 9.3 근거리 공격

- 근거리 유닛은 타겟 중심점을 기준으로 설정된 유지 거리까지 접근한다.
- 유지 거리는 0이 아니며 개발 과정에서 설정한다.
- 공격 시 공격 범위와 겹친 모든 상대 팀 유닛에게 피해를 준다.
- 현재 타겟이 아니더라도 공격 범위에 들어온 적은 피해를 받는다.
- 한 번의 공격으로 여러 적에게 동시에 피해를 줄 수 있다.

## 9.4 원거리 공격

- 원거리 유닛은 타겟과 설정된 거리를 유지한다.
- 타겟이 유지 거리보다 멀면 접근한다.
- 타겟이 유지 거리보다 가까우면 후퇴한다.
- 공격 가능한 거리에 도달하면 이동을 멈추고 총알을 발사한다.

## 9.5 총알

- 발사 시점의 타겟 방향으로 직선 이동한다.
- 발사 후에는 타겟을 추적하지 않는다.
- 설정된 투사체 속도로 이동한다.
- 설정된 투사체 수명 동안 존재한다.
- 공장, 상담실 및 다른 오브젝트를 통과한다.
- 같은 팀 유닛을 완전히 통과하며 피해를 주지 않는다.
- 원래 타겟이 아니더라도 상대 팀 유닛과 충돌하면 피해를 준다.
- 처음 충돌한 상대 팀 유닛에게 피해를 주고 즉시 사라진다.
- 아무에게도 명중하지 않으면 투사체 수명이 끝났을 때 사라진다.

## 10.1 공통 생성 규칙

유닛 생성 시 다음 데이터를 결정한다.

- 팀
- 기본 유닛 여부
- 기본 능력치
- 공격 유형
- 생성 좌표
- 생성 원인
- 적군일 경우 강화 적용 여부

## 10.2 공격 유형

각 동적 생성 유닛은 독립적으로 공격 유형을 결정한다.

```text
근거리 확률 = 50%
원거리 확률 = 50%
```

## 10.3 아군 증원

- 2일차부터 지정된 시각마다 아군 2명을 생성한다.
- 정기 증원 유닛은 기본 유닛이 아니다.
- 공격 유형만 무작위로 결정한다.
- 공격력, 방어력 등 다른 속성을 무작위로 정하지 않는다.
- 레이드 진행 또는 실패 상태에서도 생성한다.
- 레이드 실패 후 증원되더라도 이미 확정된 실패를 취소하지 않는다.

# 11. 공장 시스템

## 11.1 공장 속성

| 속성 | 설명 |
|---|---|
| 공장 종류 | 생산 시설 종류 |
| 생산 물품 | 생산 완료 시 획득하는 자원 |
| 기본 생산 시간 | 작업자 보정 전 생산 시간 |
| 계산 생산 시간 | 현재 작업자 수가 적용된 생산 시간 |
| 생산 진행 시간 | 현재 생산이 진행된 시간 |
| 생산 수량 | 생산 완료 시 증가하는 물품 수 |
| 작업 유닛 수 | 현재 작업 조건을 만족하는 아군 수 |
| 현재·최대 HP | 공장 내구도 |
| 방어력 | 적 공격 대미지 감소 |
| 점유 타일 | 공장이 차지하는 2×2 타일 |
| 파괴 여부 | 생산·작업 가능 여부 |

## 11.2 공장 종류

- 붉은 약 공장
- 푸른 약 공장
- 보라 약 공장
- 초록 약 공장
- 채굴장

채굴장은 철을 생산하지만 시스템상 공장으로 분류한다. 설치, 작업, 생산, 피해, 수리 및 게임 오버 판정에 일반 공장과 같은 규칙을 사용한다.

## 11.3 공장 작업 영역

2×2 공장 주위의 12개 타일을 작업 영역으로 사용한다.

- 위쪽 2칸
- 아래쪽 2칸
- 왼쪽 2칸
- 오른쪽 2칸
- 네 모서리의 대각선 타일 4칸

## 11.4 작업 유닛 판정

아군이 다음 조건을 모두 만족하면 작업 유닛 수에 포함한다.

1. 활동 가능한 상태다.
2. 현재 타겟이 해당 공장이다.
3. 현재 행동이 공장 작업이다.
4. 공장 주변 12개 작업 타일 안에 있다.

유닛이 이동하거나, 피격으로 전투하거나, 활동 정지하거나, 상담실에 들어가거나, 다른 명령을 받으면 즉시 작업자 수에서 제외한다.

유닛끼리 겹칠 수 있으므로 별도의 최대 작업자 수는 두지 않는다.

## 11.5 생산 조건

- 작업 유닛이 1명 이상일 때만 생산한다.
- 작업 유닛이 0명이면 생산 진행도를 유지한 채 정지한다.
- 공장이 공격받더라도 작업 조건을 만족하는 유닛이 있으면 생산한다.
- 파괴된 공장은 생산할 수 없다.
- 생산 완료 시 생산 물품을 설정된 수량만큼 공용 재고에 추가한다.
- 생산 완료 후 다음 생산을 자동으로 시작한다.

## 11.6 생산 시간

```text
작업자 보정 시간
= 기본 생산 시간 - 2 × 작업 유닛 수

계산 생산 시간
= max(공통 최소 생산 시간, 작업자 보정 시간)
```

모든 공장에는 공통 최소 생산 시간이 적용된다.

## 11.7 작업자 수 변경

작업자 수가 변하면 완료된 생산 비율을 유지한다.

```text
기존 진행률
= 기존 생산 진행 시간 / 기존 계산 생산 시간

새 생산 진행 시간
= floor(새 계산 생산 시간 × 기존 진행률)
```

예시:

```text
기존 상태: 50 / 200초
기존 진행률: 25%

새 생산 시간: 400초
새 진행 시간: floor(400 × 0.25) = 100초

변경 상태: 100 / 400초
```

## 11.8 공장 파괴

- 공장 HP가 0이 되면 파괴 상태가 된다.
- 파괴된 공장은 맵에서 사라지지 않는다.
- 점유 타일도 유지한다.
- 생산과 작업을 할 수 없다.
- 진행 중이던 생산은 0으로 초기화한다.
- 수리 후에는 생산을 처음부터 다시 시작한다.

# 12. 공장 설치·수리 시스템

## 12.1 설치 조건

- 모든 공장은 2×2 타일을 차지한다.
- 설치에는 종류별로 설정된 철 비용이 필요하다.
- 철이 부족하면 설치할 수 없다.
- 다른 시설이 점유한 타일에는 설치할 수 없다.
- 기존 공장의 2×2 점유 타일과 겹칠 수 없다.
- 기존 공장 주변의 12개 작업 타일과도 겹칠 수 없다.
- 결과적으로 다른 공장과 상하좌우 및 대각선으로 바로 붙게 설치할 수 없다.
- 상담실의 2×2 타일과 겹칠 수 없다.
- 채굴장도 같은 규칙으로 추가 설치할 수 있다.

## 12.2 수리

- 공장 수리는 철을 소비한다.
- 부분 수리는 존재하지 않는다.
- 수리하면 공장 HP를 즉시 최대치까지 회복한다.
- 수리 전 현재 HP가 속한 구간에 따라 철 비용이 달라진다.
- 현재 HP가 낮은 구간일수록 수리 비용이 높다.
- HP가 0인 공장도 수리할 수 있다.
- 수리가 완료되면 공장이 다시 작동한다.
- 생산 진행도는 초기화된 상태로 시작한다.

수리 비용 구간:

- 0~20%
- 20~40%
- 40~60%
- 60~80%
- 80~100%

정확한 구간 경계와 비용은 개발 과정에서 설정한다.

모든 공장과 채굴장이 동시에 파괴되면 즉시 게임 오버가 되므로 마지막 공장을 수리할 기회는 제공되지 않는다.

# 13. 자원·재고 시스템

| 자원 | 생산 시설 | 사용처 |
|---|---|---|
| 철 | 채굴장 | 공장 설치·수리 |
| 붉은 약 | 붉은 약 공장 | 공격력 증가 |
| 푸른 약 | 푸른 약 공장 | 방어력 증가 |
| 보라 약 | 보라 약 공장 | 공격 쿨타임 감소 |
| 초록 약 | 초록 약 공장 | HP 회복 |

## 13.2 재고

- 자원은 플레이어의 공용 재고로 관리한다.
- 공장 생산이 완료되면 해당 자원을 설정된 수량만큼 증가시킨다.
- 약 적용에 성공하면 해당 약을 1개 소비한다.
- 수량이 0이거나 아무 변화도 적용할 수 없다면 소비하지 않는다.

# 14. 약 적용 시스템

## 14.1 사용 과정

1. 인게임 상단 재고에서 사용할 약을 선택한다.
2. 적용할 아군 유닛을 클릭한다.
3. 약 보유량을 확인한다.
4. 해당 유닛에게 효과를 적용할 수 있는지 확인한다.
5. 적용 가능하면 효과를 처리하고 약 1개를 차감한다.
6. 적용할 수 없으면 약을 소비하지 않고 토스트 메시지를 표시한다.

적군, 공장, 상담 중인 유닛에는 약을 사용할 수 없다.

## 14.2 붉은 약

```text
공격력 증가량
= 현재 공격력 × (붉은 약 비율 / 100)

적용 공격력
= min(공격력 상한, 현재 공격력 + 공격력 증가량)
```

## 14.3 푸른 약

```text
방어력 증가량
= 현재 방어력 × (푸른 약 비율 / 100)

적용 방어력
= min(방어력 상한, 현재 방어력 + 방어력 증가량)
```

## 14.4 보라 약

PPT 14번 슬라이드의 `+` 표기는 감소량 표기 오류로 보고 다음 공식을 사용한다.

```text
공격 쿨타임 감소량
= 현재 공격 쿨타임 × (보라 약 비율 / 100)

적용 공격 쿨타임
= max(공격 쿨타임 하한, 현재 공격 쿨타임 - 감소량)
```

## 14.5 능력치 상한·하한

- 공격력과 방어력에는 각각 상한이 존재한다.
- 공격 쿨타임에는 하한이 존재한다.
- 약 효과가 제한을 초과하면 제한 수치까지만 적용한다.
- 일부라도 능력치가 변했다면 약을 소비한다.
- 이미 상한 또는 하한에 도달해 변화가 전혀 없다면 약을 소비하지 않고 토스트 메시지를 표시한다.

## 14.6 초록 약

```text
적용 HP
= min(100, 현재 HP + 회복량)
```

- 최대 HP를 초과한 회복량은 버린다.
- HP가 이미 100이면 사용하지 않는다.
- 약을 소비하지 않고 토스트 메시지를 표시한다.
- 활동 정지한 기본 유닛에게 사용할 수 있다.
- 회복 후 HP가 0보다 높아지면 기본 유닛을 다시 활동시킨다.
- 제거된 비기본 유닛은 회복할 수 없다.

# 15. 스트레스 시스템

## 15.1 기본 규칙

- 모든 아군 유닛은 스트레스 수치를 가진다.
- 스트레스 범위는 0~100이다.
- 스트레스 상한은 100으로 고정되며 증가시킬 수 없다.
- 스트레스가 100에 도달하면 즉시 상담 상태가 된다.
- 활동 정지 또는 상담 중인 유닛에게는 스트레스 증가를 적용하지 않는다.

## 15.2 저체력 스트레스

- 현재 HP가 30 이하이면 저체력 상태로 판정한다.
- 저체력 상태가 유지되는 동안 설정된 간격마다 스트레스가 1 증가한다.
- HP가 30을 초과하면 저체력 스트레스 증가를 중단한다.

## 15.3 작업 스트레스

- 공장에서 작업하는 동안 설정된 간격마다 스트레스가 1 증가한다.
- 공장 작업을 중단하면 작업 스트레스 증가도 중단한다.

## 15.4 중복 적용

- 저체력과 공장 작업 조건은 독립적으로 계산한다.
- 두 조건을 동시에 만족하면 두 타이머를 동시에 진행한다.
- 같은 시점에 두 타이머의 증가 시점이 도달하면 스트레스가 총 2 증가할 수 있다.
- 증가 후 스트레스는 최대 100으로 제한한다.

# 16. 상담실 시스템

## 16.1 상담실

- 상담실은 맵 왼쪽 상단에 고정 배치한다.
- 2×2 타일을 차지한다.
- 플레이어가 이동, 철거 또는 추가 설치할 수 없다.
- 상담실 타일에는 유닛이 진입하거나 통과할 수 없다.
- 상담실 타일에는 공장을 설치할 수 없다.
- 상담실은 공장이 아니다.
- 생산, 수리, 공격 및 게임 오버 판정에 포함하지 않는다.
- 적군의 공격 대상이 아니다.
- 상담 가능 인원에는 제한이 없다.
- 상담실에는 현재 상담 중인 인원수만 숫자로 표시한다.

## 16.2 상담 진입

스트레스가 100에 도달하면 다음 순서로 처리한다.

1. 상담 직전 유닛 위치를 저장한다.
2. 해당 유닛의 선택을 해제한다.
3. 기존 이동·작업·전투 명령을 중단한다.
4. 작업 중이었다면 공장 작업자 수에서 제외한다.
5. 유닛을 플레이어 카메라 밖 상담 처리 위치로 이동시킨다.
6. 선택, 이동, 작업, 공격, 피격 및 약 적용을 차단한다.
7. 활동 가능한 아군 수에서 제외한다.

상담 중인 유닛은 데이터상으로는 남아 있지만 게임 판정에서는 사망한 유닛처럼 취급한다. 따라서 모든 아군이 활동 정지, 제거 또는 상담 상태라면 레이드 실패 조건을 만족한다.

## 16.3 상담 진행

- 상담 중에는 설정된 간격마다 스트레스가 1 감소한다.
- 스트레스가 0이 되어야만 상담을 종료할 수 있다.

## 16.4 상담 복귀

1. 상담 상태를 해제한다.
2. 상담 직전에 저장한 위치로 즉시 순간이동한다.
3. 현재 HP가 30 이하라면 HP를 31로 변경한다.
4. HP가 31 이상이면 기존 HP를 유지한다.
5. 다시 활동 가능한 아군 수와 다른 게임 판정에 포함한다.

# 17. 레이드 시스템

## 17.1 레이드 구조

- 적군은 레이드를 통해서만 생성된다.
- 적이 존재하는 동안 하나의 레이드 상태가 유지된다.
- 레이드 시각이 다시 되어도 별도의 새 레이드를 생성하지 않는다.
- 기존 레이드에 추가 적 병력이 합류한다.
- 레이드 발생 횟수는 병력 생성 이벤트를 의미하며, 진행 중인 레이드 자체를 초기화하지 않는다.

## 17.2 적 생성 수

```text
초당 적 생성 수
= floor(
    기본 초당 적 생성 수
    + 현재 일차 / 생성 수 증가 계수
)
```

## 17.3 적 생성 지속시간

PPT의 `기본 생산 시간`은 `기본 적 생성 지속시간`으로 정정한다.

```text
적 생성 지속시간
= floor(
    기본 적 생성 지속시간
    + 현재 일차 / 생성시간 증가 계수
)
```

## 17.4 총생성 수

```text
총생성 수
= 초당 적 생성 수 × 적 생성 지속시간
```

각 적은 생성 기간 동안 설정된 간격에 맞춰 오른쪽 생성 구역에서 등장한다.

## 17.5 적 공격 유형

각 적의 공격 유형은 독립적으로 판정한다.

```text
근거리 확률 = 50%
원거리 확률 = 50%
```

## 17.6 적 능력치 강화

각 적은 공격력, 방어력, 공격 쿨타임 강화 여부를 독립적으로 판정한다.

```text
공격력 강화 확률
= 기본 공격력 강화 확률 + 현재 일차 - 고정 차감값

방어력 강화 확률
= 기본 방어력 강화 확률 + 현재 일차 - 고정 차감값

공격 쿨타임 강화 확률
= 기본 쿨타임 강화 확률 + 현재 일차 - 고정 차감값
```

- 각 강화 확률에는 개별 상한이 존재한다.
- 공격력과 방어력은 아군 강화 약과 같은 증가 함수를 사용한다.
- 공격 쿨타임은 보라 약과 같은 감소 함수를 사용한다.
- 강화 적용 후에도 능력치 상한과 쿨타임 하한을 적용한다.

## 17.7 추가 강화 판정

- 기본 강화 판정 후 추가 강화 여부를 한 번 판정한다.
- 추가 적용 확률은 일차에 따라 증가한다.
- 추가 적용 확률에도 상한이 존재한다.
- 성공하면 최초 강화 처리와 같은 함수를 한 번 더 실행한다.
- 추가 강화 판정은 적 하나당 최대 한 번만 실행한다.

# 18. 레이드 성공·실패 규칙

## 18.1 레이드 성공

다음 조건을 모두 만족하면 레이드 성공이다.

1. 적이 1명 이상 존재한 적이 있다.
2. 현재 진행 중인 적 생성이 모두 끝났다.
3. 정상적인 전투로 적 수가 0이 됐다.
4. 현재 레이드가 실패 상태가 아니다.

적 생성 중 현재 적 수가 잠시 0이 되더라도 성공하지 않는다.

레이드 성공에 따른 별도 보상은 현재 정의되지 않았다.

## 18.2 레이드 실패

활동 가능한 아군이 한 명도 없으면 레이드 실패다.

다음 유닛은 활동 가능한 아군으로 계산하지 않는다.

- HP 0으로 활동 정지한 기본 유닛
- 제거된 비기본 유닛
- 상담 중인 유닛

레이드 실패는 게임 오버가 아니다.

## 18.3 실패 확정 후 처리

1. 레이드 실패 상태를 확정한다.
2. 적 퇴각 대기시간을 시작한다.
3. 대기시간 동안 적은 계속 공장을 공격한다.
4. 이때 적의 타겟은 공장으로 고정한다.
5. 아군이 정기 증원되거나 상담을 마치고 복귀해도 실패 상태를 취소하지 않는다.
6. 증원된 아군이 존재해도 적은 공장을 계속 공격한다.
7. 레이드 시각이 다시 되어 적 병력 생성이 시작되면 퇴각 대기시간을 한 번 초기화한다.
8. 개별 적이 한 명씩 생성될 때마다 초기화하지는 않는다.
9. 퇴각 대기시간이 끝나면 모든 적이 원래 생성된 오른쪽 생성 구역으로 이동한다.
10. 생성 구역에 도착한 적은 게임에서 제거한다.

## 18.4 퇴각 중 레이드 재발동

적이 물리적으로 생성 구역으로 퇴각하는 중 레이드 시각이 되면 다음과 같이 처리한다.

1. 기존 적의 퇴각을 취소한다.
2. 추가 적 병력을 생성한다.
3. 기존 적과 새 적이 다시 공장을 공격한다.
4. 퇴각 대기시간을 다시 계산한다.

## 18.5 강제 제거 판정

레이드 실패 후 퇴각으로 제거된 적은 다음에 포함하지 않는다.

- 레이드 성공
- 적 처치 수
- 처치 보상

레이드 실패 보상도 존재하지 않는다.

# 19. 게임 오버 규칙

## 19.1 게임 오버 조건

일반 공장과 채굴장을 포함한 모든 공장의 HP가 0이면 즉시 게임 오버다.

- 상담실은 공장 수에 포함하지 않는다.
- 아군 전멸만으로는 게임 오버가 발생하지 않는다.
- 마지막 공장이 파괴되면 수리 기회를 제공하지 않는다.

## 19.2 게임 오버 처리

1. 게임 시간 진행을 정지한다.
2. 모든 유닛, 공장 및 레이드 처리를 정지한다.
3. 플레이어의 인게임 조작을 차단한다.
4. 게임 오버 결과 화면을 표시한다.
5. 플레이어가 메인 화면 또는 처음부터를 선택할 때까지 현재 화면을 유지한다.

## 19.3 결과 정보

표시 후보는 다음과 같다.

- 도달 일차
- 사망한 아군 수
- 정상적으로 처치한 적군 수
- 종류별 총생산 자원
- 누적 획득 아군 수
- 현재 생존·활동 정지·상담·사망 아군 수

기본 유닛의 활동 정지와 상담 상태는 사망 수에 포함하지 않는다. 강제 퇴각으로 제거된 적은 처치 수에 포함하지 않는다.

결과 통계는 개발 우선순위에 따라 일부 또는 전체를 제외할 수 있다.

# 21. 저장·아웃게임 시스템

## 21.1 저장 정책

- 최신 저장 데이터 하나만 사용한다.
- 저장 슬롯 선택 기능은 없다.
- 별도의 불러오기 버튼이나 저장 목록 화면은 제공하지 않는다.
- 새로 저장하면 기존 최신 저장 데이터를 갱신한다.

## 21.2 저장 대상

최소한 다음 데이터를 저장해야 한다.

- 현재 일차와 시각
- 시간 이벤트 진행 상태
- 아군과 적군의 종류, 위치, 능력치 및 행동 상태
- 기본 유닛·활동 정지·상담 상태
- 유닛별 스트레스와 상담 진행도
- 공장의 종류, 위치, HP 및 파괴 상태
- 공장별 작업자와 생산 진행도
- 철과 약 재고
- 레이드 진행·실패·퇴각 상태
- 적 생성 진행 상태
- 누적 결과 통계

## 21.3 메인 화면

현재 확정된 기능은 다음과 같다.

- **이어하기:** 최신 저장 데이터를 불러와 게임을 계속한다.
- **처음부터:** 초기 게임 상태로 새 게임을 시작한다.
- **게임 종료:** 애플리케이션을 종료한다.

저장 데이터가 없다면 이어하기를 비활성화한다.

과거 저장 데이터를 선택하는 불러오기 기능은 존재하지 않는다.

## 21.4 미완성 아웃게임 기능

다음 항목은 추후 세부 설계가 필요하다.

- 저장 시점과 저장 방식
- 처음부터 선택 시 기존 저장 데이터 처리
- 일시정지 화면
- 설정
- 게임 종료 확인
- 튜토리얼
- 메인 화면 및 결과 화면 UI

- 적이 아군과 공장을 함께 거리 비교해 타겟을 고르는 규칙
- 공장을 공격하던 적이 아군에게 피격되면 타겟을 바꾸는 규칙
- 작업 중 피격된 아군이 공격자를 처치한 뒤 즉시 기존 작업으로 복귀하는 전체 처리 순서
- 근거리 공격의 다중 피격 방식
- 총알의 속도·수명·첫 적 명중 후 제거
- 총알이 아군과 오브젝트를 통과하는 정확한 규칙
- 공장 방어력과 공장 대미지 계산

## 공장·생산

- 작업 유닛으로 인정되는 네 가지 조건
- 작업자 수가 0일 때 생산 진행도를 유지하며 멈추는 규칙
- 작업자 수 변경 시 사용하는 정확한 진행도 변환식
- 생산 완료 후 재고 증가와 자동 재생산
- 파괴 시 생산 진행도가 초기화되는 규칙
- 파괴된 공장이 타일을 계속 점유하는 규칙
- HP 0인 공장의 수리 가능 여부
- 마지막 공장 파괴 시 수리 없이 즉시 게임 오버가 되는 규칙

## 약·자원

- 약 재고가 0일 때 실패 처리
- 상한을 일부 초과하면 상한까지만 적용하는 규칙
- 아무 변화가 없을 때 약을 소비하지 않는 규칙
- 회복 약으로 기본 유닛을 재가동하는 규칙
- 상담 중 유닛에게 약을 사용할 수 없는 규칙

## 스트레스·상담

- 저체력과 작업 스트레스 타이머의 독립 계산
- 두 타이머가 동시에 끝나면 스트레스가 2 증가할 수 있는 규칙
- 상담 진입 시 선택·작업·전투에서 제외되는 처리
- 상담 중 유닛이 레이드 판정에서 사망 취급되는 규칙
- 실제 유닛은 상담실이 아니라 카메라 밖에 배치되는 규칙
- 상담 완료 후 기존 위치로 순간이동하는 규칙

## 레이드

- 적 생성 중 적이 잠시 0명이어도 성공하지 않는 조건
- 강화 능력치별 독립 확률 판정
- 추가 강화는 최대 한 번만 수행하는 규칙
- 레이드 실패 후 적의 타겟이 공장으로 고정되는 규칙
- 아군 증원이나 상담 복귀로 실패가 취소되지 않는 규칙
- 레이드 재발동 시 퇴각 대기시간을 한 번만 초기화하는 규칙
- 물리적 퇴각 중 레이드가 재발동하면 퇴각을 취소하는 규칙
- 강제 제거된 적이 성공·처치·보상에 포함되지 않는 규칙

## 저장·결과

- 저장해야 하는 개별 게임 상태
- 저장 슬롯과 과거 저장 선택 기능이 없다는 정확한 의미
- 게임 오버 결과 통계의 계산 기준
- 활동 정지·상담 유닛이 사망 수에는 포함되지 않는 규칙

## UI

- 체력바 색상 변화 방향
- 유닛과 공장의 마우스 오버 정보
- 약 적용 실패 및 자원 부족 토스트 조건
- 상담 인원수 표시 방식

이 문서를 상세 기획의 기준으로 사용하고, PPT는 팀 발표와 전체 구조 파악용 요약본으로 사용하는 것이 적절하다.
